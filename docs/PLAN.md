# 實作計畫 (Implementation Plan)

將經典 Tamagotchi 機制加入現有網頁版電子雞的改造計畫。

---

## 目前狀態

現有功能：
- 蛋形機殼 + 銀色螢幕框 + 火焰裝飾
- 小雞角色（16×16 像素精靈圖，10 種動畫狀態）
- 3 按鈕（A/B/C）+ 彈出式選單
- 4 項百分比屬性條（HP/FD/EN/CL）
- 手動睡覺/餵食/玩耍/清潔/治療
- localStorage 存檔 + 離線衰減

---

## 改造目標

將遊戲從簡化版升級為忠實還原經典 Tamagotchi P1 的完整機制。

---

## 階段規劃

### 第一階段：核心屬性改造

**目標**：將屬性系統從百分比改為經典愛心制，加入 8 圖標導航

**檔案修改**：
- `index.html` — 移除舊 status-area 和 action-menu，加入 LCD 內愛心顯示、overlay 容器
- `style.css` — 愛心顯示樣式、LCD overlay 樣式、圖標高亮、關燈效果
- `game.js` — 屬性改為 0-4 愛心、8 圖標導航邏輯、體重系統、正餐/零食區分

**具體項目**：
1. HTML 結構重構（移除 status-area，加入 hearts-bar 和 overlay）
2. 愛心制屬性（hunger 0-4, happiness 0-4）
3. 體重系統（正餐 +1g, 零食 +2g, 遊戲 -1g）
4. 正餐/零食子選單
5. 8 圖標導航（A=下一個, B=執行, C=取消）
6. 狀態查看畫面（顯示年齡/體重/紀律/飢餓/開心）

### 第二階段：進化系統

**目標**：實作 10 個角色的精靈圖和進化路線

**檔案修改**：
- `game.js` — 10 個角色精靈圖、角色數據表（衰減速率/睡眠時間/基礎體重）、進化邏輯

**具體項目**：
1. 10 個角色的 16×16 精靈圖設計
2. 角色數據表（CHARACTERS 物件）
3. 進化判定邏輯（照顧失誤 + 紀律值）
4. 進化動畫（閃爍效果）
5. 依角色調整衰減速率

### 第三階段：照顧系統

**目標**：加入紀律、便便、睡眠排程、照顧失誤

**檔案修改**：
- `game.js` — 紀律系統、便便計時、真實時鐘睡眠、15 分鐘照顧失誤計時、注意圖標

**具體項目**：
1. 紀律值（0/25/50/75/100%）
2. 管教呼叫（每階段 4 次機會）
3. 便便系統（計時產生、最多 4 坨、滿了生病）
4. 便便 Canvas 繪製
5. 真實時鐘睡眠（依角色就寢/起床時間）
6. 關燈/開燈功能
7. 照顧失誤計數（15 分鐘計時器）
8. 注意圖標閃爍

### 第四階段：迷你遊戲 + 收尾

**目標**：猜方向迷你遊戲、死亡系統完善、離線計算修正

**檔案修改**：
- `game.js` — 迷你遊戲邏輯、死亡條件完善、離線計算
- `CLAUDE.md` — 更新文件
- `README.md` — 更新文件
- `docs/TASKS.md` — 更新完成狀態

**具體項目**：
1. 猜方向迷你遊戲（5 回合，LCD overlay 顯示）
2. 死亡條件完善（老死/餓死/病死/失誤死）
3. 各角色最大壽命
4. 離線計算修正（依新的愛心制衰減計算）
5. 更新 CLAUDE.md 和 README.md
6. 完整測試

## 技術注意事項

### 精靈圖策略
- 每個角色定義 1 個基礎精靈圖（normal 狀態）
- 睡覺/生病/死亡通過程式修改渲染（眼睛替換、顏色灰化）
- 減少精靈圖定義量，增加程式碼靈活性

### 狀態機設計
```
UI 模式:
  main        → 顯示寵物
  icon-select → 8 圖標選擇中
  feed-menu   → 正餐/零食子選單
  game        → 迷你遊戲進行中
  status      → 查看狀態畫面
  attention   → 查看注意資訊
```

### 計時器架構
```
每 500ms  → 動畫幀切換
每 1min   → 檢查飢餓/開心衰減、便便、睡眠、進化
每 10s    → 自動存檔
每 1s     → 檢查照顧失誤計時（15 分鐘）
```

### 向下相容
- 舊版 localStorage 存檔格式不相容新版
- 首次載入新版時清除舊存檔，重新開始
