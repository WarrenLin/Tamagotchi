# 遊戲規格書 (Game Specification)

基於經典 Tamagotchi P1 (1996) 的遊戲機制研究，定義網頁版實作規格。

---

## 一、成長階段與進化系統

### 1.1 階段總覽

| 階段 | 角色 | 持續時間 | 基礎體重 |
|------|------|----------|----------|
| 寶寶 Baby | Babytchi | 65 分鐘 | 5g |
| 幼兒 Child | Marutchi | 至年齡 3（約 2 天） | 10g |
| 青少年 Teen | Tamatchi（好）/ Kuchitamatchi（差） | 至年齡 6（約 3 天） | 20g |
| 成體 Adult | 6 種角色（見 1.3） | 可存活至 21-25 天 | 依角色 |

### 1.2 年齡系統

- 1 現實天 = 1 電子雞歲
- 每次從睡眠中醒來，年齡 +1
- 最大壽命約 21-25 歲，照顧品質越好越長壽

### 1.3 進化路線（P1）

```
Babytchi (65分鐘後)
  └→ Marutchi
       ├─ 0-1 次照顧失誤 → Tamatchi（好青少年）
       │    ├─ 100% 紀律 → Mametchi ⭐（最佳）
       │    ├─  75% 紀律 → Ginjirotchi
       │    └─ 0-50% 紀律 → Maskutchi
       │
       └─ 2+ 次照顧失誤 → Kuchitamatchi（差青少年）
            ├─ 100% 紀律 → Kuchipatchi
            ├─  中等紀律 → Nyorotchi
            └─  低紀律   → Tarakotchi（最差）
```

### 1.4 進化決定因素

| 因素 | 說明 |
|------|------|
| 照顧失誤次數 | 幼兒期 0-1 次 → 好路線；2+ 次 → 差路線 |
| 紀律值 | 青少年期的紀律值決定成體角色 |
| 紀律延續 | 幼兒期 75-100% → 青少年從 50% 開始；0-50% → 從 0% 開始 |

## 三、操作功能（8 圖標）

### 3.1 圖標配置

| 位置 | 圖標 | 功能 |
|------|------|------|
| 上排 1 | 🍴 | 餵食（正餐/零食子選單） |
| 上排 2 | 💡 | 開/關燈 |
| 上排 3 | ⚾ | 玩遊戲（猜方向迷你遊戲） |
| 上排 4 | 💉 | 吃藥（治療生病） |
| 下排 1 | 🛁 | 清潔（清除便便） |
| 下排 2 | 📢 | 管教（紀律 +25%） |
| 下排 3 | 📊 | 查看狀態（年齡/體重/紀律/飢餓/開心） |
| 下排 4 | ⚠️ | 注意（查看寵物需求） |

### 3.2 按鈕操控

| 按鈕 | 主畫面 | 圖標選擇中 | 子選單中 | 迷你遊戲中 |
|------|--------|-----------|---------|-----------|
| A | 選擇第一個圖標 | 下一個圖標 | 切換選項 | 猜「左」 |
| B | 無 | 執行功能 | 確認 | 猜「右」 |
| C | 無 | 取消選擇 | 返回 | 退出遊戲 |

鍵盤對應：A=A/←、B=S/Enter/↓、C=D/Esc/→

### 3.3 各操作效果

#### 餵食
| 類型 | 飢餓 | 開心 | 體重 | 備註 |
|------|------|------|------|------|
| 正餐 Meal | +1 ♥ | — | +1g | 飽了(4♥)會拒吃 |
| 零食 Snack | — | +1 ♥ | +2g | 永遠不會拒吃 |

#### 迷你遊戲（猜方向）
- 5 回合，猜寵物轉左或轉右
- 猜對 ≥3 次：開心 +1♥，體重 -1g
- 全對（5/5）：開心 +2♥，體重 -1g
- 猜錯多數：無獎勵，體重 -1g

#### 管教
- 寵物「裝哭」時使用（飢餓和開心都不是 0 但發出呼叫）
- 成功管教：紀律 +25%
- 每階段最多 4 次管教呼叫機會

## 四、照顧失誤系統

### 4.1 觸發條件

以下情況超過 **15 分鐘** 未處理 = 1 次照顧失誤：

| 情況 | 處理方式 |
|------|---------|
| 飢餓降到 0♥ | 餵食 |
| 開心降到 0♥ | 玩遊戲或餵零食 |
| 寵物睡著未關燈 | 選擇燈光圖標關燈 |
| 生病未治療 | 選擇吃藥圖標 |

### 4.2 影響

- 照顧失誤**永遠無法撤銷**，累積影響進化路線
- 幼兒期 0-1 次 → 進化為好青少年
- 幼兒期 2+ 次 → 進化為差青少年
- 成體階段累積 ~5 次 → 死亡
- 每次失誤縮短壽命

## 五、便便系統

### 5.1 便便頻率

| 階段 | 頻率 |
|------|------|
| 寶寶 | 出生後 15 分鐘第一次，40 分鐘第二次 |
| 幼兒 | 進化後 5 分鐘，之後每 3 小時 |
| 青少年/成體 | 約每 3 小時 |

### 5.2 規則

- 螢幕最多堆積 **4 坨** 便便
- 4 坨滿了 → 自動生病
- 使用清潔圖標（🛁）清除所有便便

## 六、睡眠系統

### 6.1 各角色睡眠時間

| 角色 | 就寢 | 起床 |
|------|------|------|
| Babytchi | 無固定（出生 40 分鐘後小睡） | — |
| Marutchi | 20:00 | 09:00 |
| Tamatchi | 21:00 | 09:00 |
| Kuchitamatchi | 21:00 | 09:00 |
| Mametchi | 22:00 | 09:00 |
| Ginjirotchi | 22:00 | 09:00 |
| Maskutchi | 23:00 | 11:00 |
| Kuchipatchi | 22:00 | 09:00 |
| Nyorotchi | 22:00 | 09:00 |
| Tarakotchi | 22:00 | 10:00 |

### 6.2 規則

- 到就寢時間自動入睡
- 玩家必須在 **15 分鐘內關燈**，否則算照顧失誤
- 到起床時間自動起床
- 睡眠中不需要任何照顧

## 七、生病與死亡

### 7.1 生病原因

| 原因 | 說明 |
|------|------|
| 便便堆積 4 坨 | 清潔度過差 |
| 每階段固定 1 次 | 成長過程自然生病 |
| 幼兒吃零食 3+ 個 | 過量餵零食 |

### 7.2 治療

- 使用吃藥圖標（💉）
- 可能需要 **1-3 劑** 才能痊癒

### 7.3 死亡條件

| 條件 | 說明 |
|------|------|
| 自然老死 | 達到最大壽命（21-25 天） |
| 累積照顧失誤 | 成體期約 5 次失誤 |
| 餓死 | 飢餓+開心同時 0♥ 持續 12+ 小時 |
| 病死 | 生病後長時間未治療 |

## 八、角色精靈圖需求

每個角色需要 1 個 16×16 基礎精靈圖，共 10 個角色：

| # | 角色名 | 階段 | 視覺特徵 |
|---|--------|------|---------|
| 1 | Babytchi | 寶寶 | 極小圓形，只有眼睛 |
| 2 | Marutchi | 幼兒 | 圓球形，眼睛+小腳 |
| 3 | Tamatchi | 好青少年 | 圓身+明顯腿部 |
| 4 | Kuchitamatchi | 差青少年 | 突出的嘴/喙 |
| 5 | Mametchi | 最佳成體 | 頭上有耳/帽形突起 |
| 6 | Ginjirotchi | 良好成體 | 光滑圓形，平頂 |
| 7 | Maskutchi | 普通成體 | 眼部有面具帶 |
| 8 | Kuchipatchi | 差線良好 | 很圓，大嘴 |
| 9 | Nyorotchi | 差成體 | 蛇/蟲形，細長 |
| 10 | Tarakotchi | 最差成體 | 凌亂的團狀 |

### 狀態修改（程式碼處理，不需額外精靈圖）

| 狀態 | 處理方式 |
|------|---------|
| 睡覺 | 眼睛替換為水平線 + Zzz |
| 生病 | 身體顏色灰化 + 骷髏圖標 |
| 死亡 | 眼睛替換為 X + 灰化 |
| 開心彈跳 | Y 軸偏移動畫 |
| 吃東西 | 食物圖標出現在嘴邊 |

## 十、Web 版調整

以下為相對原版的調整，使遊戲更適合網頁體驗：

| 項目 | 原版 | Web 版 |
|------|------|--------|
| 寶寶期 | 65 分鐘 | 保持 65 分鐘 |
| 年齡計算 | 每次醒來 +1 | 保持（基於真實時鐘） |
| 音效 | 蜂鳴器 | 無（純視覺） |
| 螢幕 | 32×16 LCD | 16×16 精靈 + HTML overlay |
| 按鈕 | 3 個實體按鈕 | 3 個按鈕 + 鍵盤 + 點擊圖標 |
